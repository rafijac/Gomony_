# Gomony Keyboard Navigation & Focus Management Plan (Final)

## Value Statement
As a player or spectator, I want to navigate and operate all Gomony UI elements using only the keyboard, with clear focus indicators and screen reader support, so that the game is fully accessible and inclusive for all users and roles.

## Objective
Deliver comprehensive keyboard navigation, focus management, ARIA roles, and accessibility features for all interactive elements (board, dialogs, controls, spectate mode), with explicit test coverage for all user roles and edge cases.

## Requirements
- Tab order and focus management for all interactive elements (board, dialogs, controls)
- ARIA roles and live regions
- Keyboard shortcuts
- Spectate mode accessibility (read-only, summary-only modals for spectators)
- Edge case scenarios for keyboard/focus flows
- Test coverage for all user roles and scenarios
- Avatars/names editable only in lobby, locked after game start
- Win/loss UI supports standard and optional custom messages
- Backend sync for avatars/names
